# Architecture Decision Records

This directory contains Architecture Decision Records for the project.

## Naming

Use sequential numbering:

```text
0001-decision-title.md
0002-decision-title.md
0003-decision-title.md
```

## Status values

- Proposed
- Accepted
- Superseded
- Rejected

## When to create an ADR

Create an ADR when a decision affects long-term architecture, including:

- Sanity content model structure
- GROQ query strategy
- Next.js rendering strategy
- SSG, ISR, rebuild, or preview mode decisions
- SEO metadata strategy
- Image pipeline
- Deployment architecture
- Major dependency choices
- Cross-cutting component architecture

## Template

Use `0000-template.md` as the starting point.
