# Project Plans

This directory contains implementation plans for non-trivial work.

## Naming

Use date-prefixed names:

```text
YYYY-MM-DD-feature-or-task-name.md
```

## Status values

- Draft
- Approved
- In progress
- Done
- Paused
- Cancelled

## When to create a plan

Create a plan when work includes multiple steps, multiple files, architectural risk, schema changes, or deployment impact.

Use `0000-template.md` as the starting point.
